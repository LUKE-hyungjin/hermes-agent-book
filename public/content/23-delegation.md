# 23. 서브에이전트 위임

> `delegate_task`는 현재 대화에서 자식 AIAgent를 띄워 별도 맥락과 도구셋으로 작업을 나누는 기능입니다. 중요한 점은 "보조 창 하나 더"가 아니라 "완전히 새 대화 하나 더"라는 점입니다.

## `delegate_task`가 하는 일

공식 문서에 따르면 자식 에이전트는 다음 조건에서 실행됩니다.

- 부모와 분리된 컨텍스트
- 제한된 도구셋
- 독립된 터미널 세션
- 부모 컨텍스트로는 최종 요약만 반환

즉 자식이 중간에 어떤 파일을 읽고 어떤 도구를 썼는지는 부모 문맥을 거의 오염시키지 않습니다. 대규모 조사나 병렬 작업에 특히 유리한 이유가 여기에 있습니다.

## 기본 사용 예

### 단일 작업

```python
delegate_task(
    goal="테스트가 왜 실패하는지 디버깅",
    context="test_foo.py 42번째 줄 assertion 실패",
    toolsets=["terminal", "file"],
)
```

### 병렬 배치

문서는 최대 3개의 서브에이전트를 동시에 돌릴 수 있다고 설명합니다.

```python
delegate_task(tasks=[
    {"goal": "주제 A 조사", "toolsets": ["web"]},
    {"goal": "주제 B 조사", "toolsets": ["web"]},
    {"goal": "빌드 고치기", "toolsets": ["terminal", "file"]},
])
```

## 가장 중요한 원칙: 서브에이전트는 아무것도 모른다

문서에서 가장 강하게 강조하는 경고가 이것입니다. 자식 에이전트는 부모 대화 이력을 전혀 이어받지 않습니다. 부모가 방금 본 오류, 방금 읽은 파일, 직전 도구 출력도 자동 전달되지 않습니다.

### 나쁜 예

```python
delegate_task(goal="그 에러 좀 고쳐")
```

이 상태에서 자식은 "어떤 에러인지"조차 모릅니다.

### 좋은 예

```python
delegate_task(
    goal="api/handlers.py의 TypeError 수정",
    context="""
api/handlers.py 47번째 줄에서
'NoneType' object has no attribute 'get' 예외가 난다.
process_request()는 parse_body()의 dict 결과를 기대하지만
Content-Type이 없으면 parse_body()가 None을 반환한다.
프로젝트는 /home/user/myproject 이고 Python 3.11을 사용한다.
""",
)
```

핵심은 목표와 맥락을 충분히 써서, 자식이 부모에게 재질문하지 않아도 되게 만드는 것입니다.

## 실전 사용 패턴

문서는 다음 같은 사례를 제시합니다.

| 패턴 | 설명 |
|------|------|
| 병렬 조사 | 서로 다른 주제를 동시에 웹 조사 |
| 리뷰 + 수정 | 새 맥락에서 보안 점검 후 수정 |
| 대규모 리팩터링 | 부모 문맥을 오염시키지 않고 일괄 수정 |

예를 들어 "웹 조사 3건 병렬", "인증 모듈 점검 후 pytest", "여러 파일의 `print()`를 logging으로 교체" 같은 작업이 대표적입니다.

## 언제 위임이 좋은가

| 상황 | 위임 적합성 |
|------|-------------|
| 서로 독립적인 조사 과제 여러 개 | 매우 높음 |
| 한 에이전트 문맥이 너무 커지는 리팩터링 | 높음 |
| 간단한 단일 파일 수정 | 낮음 |
| 부모와 상시 상호작용이 필요한 작업 | 낮음 |

## 위임 품질을 높이는 팁

1. `goal`은 구체적인 산출물 중심으로 쓴다.
2. `context`에는 파일 경로, 오류 내용, 실행 명령, 환경 정보를 넣는다.
3. 도구셋은 최소 권한으로 준다.
4. 병렬로 돌릴 때는 과제가 진짜 독립적인지 먼저 본다.

:::tip
서브에이전트는 "알아서 잘하겠지"가 아니라 "잘할 수 있게 입력을 설계해야 하는 대상"입니다. 위임 품질은 결국 맥락 전달 품질에 달려 있습니다.
:::

## 부모 컨텍스트를 왜 아끼나

이 기능의 실제 가치 중 하나는 부모 대화창을 덜 어지럽히는 데 있습니다. 자식이 수십 번의 도구 호출을 하더라도 부모는 마지막 요약만 받으므로, 장기 세션에서 맥락 예산을 아끼는 데 도움이 됩니다.

# 08. 세션 관리

> Hermes는 모든 대화를 자동으로 세션으로 저장하므로, 작업을 끊었다가도 문맥을 거의 그대로 이어갈 수 있습니다.

## 세션이란 무엇인가요?

CLI에서 시작한 대화든, Telegram이나 Discord에서 이어지는 채팅이든 Hermes는 각각을 세션으로 저장합니다. 세션은 단순 로그가 아니라, 나중에 다시 불러와 검색하고 이어서 작업할 수 있는 **대화 단위의 작업 기록**입니다.

## 저장 구조

Hermes는 세션을 두 가지 방식으로 함께 보관합니다.

| 저장소 | 위치 | 역할 |
|--------|------|------|
| SQLite DB | `~/.hermes/state.db` | 메타데이터와 전문 검색(FTS5) |
| JSONL 기록 | `~/.hermes/sessions/` | 원본 대화와 도구 호출 로그 |

SQLite에는 다음 정보가 들어갑니다.

- 세션 ID
- 소스 플랫폼(`cli`, `telegram`, `discord`, `whatsapp`, `slack`)
- 사용자 ID
- 세션 제목
- 사용한 모델과 설정
- 시스템 프롬프트 스냅샷
- 메시지/도구 호출/도구 결과
- 토큰 수
- 시작 및 종료 시각

## 세션 다시 열기

가장 자주 쓰는 복구 방식은 `--continue`와 `--resume`입니다.

```bash
# 가장 최근 CLI 세션 이어서
hermes --continue
hermes -c

# 특정 세션 ID로 복구
hermes --resume 20250305_091523_a1b2c3d4
hermes -r 20250305_091523_a1b2c3d4

# 제목으로 복구
hermes --resume "refactoring auth"
```

| 방식 | 장점 | 적합한 경우 |
|------|------|-------------|
| `-c` | 가장 빠름 | 방금 하던 작업 재개 |
| `-r <id>` | 가장 정확함 | 오래된 세션 재현 |
| `-c "제목"` | 제목 계열 최신본 자동 선택 | 반복 프로젝트 |

세션 ID 형식은 `YYYYMMDD_HHMMSS_<8자리16진수>`입니다. 예를 들어 `20250305_091523_a1b2c3d4`처럼 보입니다.

## 세션 복구 화면

세션을 이어서 열면 Hermes는 입력 프롬프트 전에 "이전 대화 요약" 패널을 보여 줍니다. 이 패널은 최근 사용자 메시지와 어시스턴트 응답만 압축해서 표시하고, 도구 호출은 `terminal`, `web_search` 같은 이름 목록으로 묶어서 보여 줍니다.

기본 정책은 다음과 같습니다.

| 항목 | 기본 동작 |
|------|-----------|
| 최근 교환 수 | 마지막 10개 내외 |
| 사용자 메시지 | 300자 안팎으로 축약 |
| 어시스턴트 메시지 | 200자 또는 3줄 정도로 축약 |
| 시스템/내부 추론 | 숨김 |
| 도구 결과 전문 | 숨김 |

간단 표시만 원하면 아래처럼 바꿀 수 있습니다.

```yaml
display:
  resume_display: minimal
```

:::tip
긴 세션을 자주 이어서 쓰는 사용자라면 `/title`로 세션 이름을 먼저 지정해 두세요. 나중에 `hermes -c "프로젝트명"`처럼 매우 빠르게 돌아올 수 있습니다.
:::

## 세션 제목

Hermes는 첫 번째 왕복 이후 자동으로 3~7단어 정도의 짧은 제목을 생성합니다. 별도 지연 없이 백그라운드에서 만들어지므로 체감 성능에 거의 영향이 없습니다. 물론 수동 제목도 줄 수 있습니다.

```text
/title 인증 리팩터링
```

이후 같은 프로젝트에서 파생된 세션이 생기면 `my project #2`, `#3`처럼 계보가 붙을 수 있고, `-c "my project"`는 보통 그중 최신본을 복구합니다.

## 왜 세션이 중요한가요?

Hermes의 강점 중 하나는 "한 번에 다 끝내지 않아도 된다"는 점입니다. 예를 들어 오전에는 저장소 분석만 시키고, 오후에는 같은 세션을 다시 열어 테스트 수정과 리팩터링을 이어갈 수 있습니다.

```bash
hermes -c "프론트엔드 성능 점검"
```

이렇게 하면 이전에 어떤 파일을 열었는지, 어떤 도구를 썼는지, 어디까지 작업했는지를 다시 설명할 필요가 크게 줄어듭니다.

## 운영 팁

| 팁 | 이유 |
|----|------|
| 큰 작업은 `/title`로 이름 지정 | 검색과 복구가 쉬워짐 |
| 실험 세션과 본작업 세션 분리 | 문맥 오염 방지 |
| 토큰이 커진 세션은 적절히 압축 | 비용과 품질 관리 |
| 메시징 플랫폼별 세션 특성 이해 | 채널/스레드별 흐름 차이 대응 |

:::info
세션은 단순 대화 저장이 아니라, Hermes가 장기 작업을 현실적으로 수행하게 만드는 기반 기능입니다. CLI를 자주 쓴다면 세션 복구 습관만으로도 생산성이 크게 달라집니다.
:::

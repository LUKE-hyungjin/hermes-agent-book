# 06. CLI 인터페이스

> Hermes CLI는 웹 UI가 아니라 터미널에 최적화된 TUI이며, 대화·도구 실행·세션 복구를 한 화면에서 처리합니다.

## CLI가 중요한 이유

Hermes Agent의 중심은 여전히 CLI입니다. 메시징 플랫폼이나 API 서버도 강력하지만, 코드 읽기, 명령 실행, 파일 수정, 세션 재개, 음성 입력, 도구 스트리밍까지 가장 완전한 경험은 터미널에서 얻습니다. 특히 개발자에게는 브라우저를 벗어나지 않아도 된다는 점이 큰 장점입니다.

## 기본 실행 패턴

가장 많이 쓰는 형태는 아래와 같습니다.

```bash
# 기본 대화형 세션
hermes

# 한 번만 질의하고 종료
hermes chat -q "README를 요약해줘"

# 특정 모델/프로바이더 강제
hermes chat --provider openrouter --model anthropic/claude-sonnet-4.6

# 특정 도구셋만 허용
hermes chat --toolsets "web,terminal,skills"

# 이전 세션 이어서 작업
hermes --continue
hermes --resume 20250305_091523_a1b2c3d4
```

| 옵션 | 용도 | 추천 상황 |
|------|------|-----------|
| `-q`, `--query` | 비대화형 1회 실행 | 스크립트, 빠른 점검 |
| `--provider` | 추론 제공자 고정 | 디버깅, 비용 통제 |
| `--toolsets` | 사용 도구 범위 제한 | 안전성 확보, 재현성 |
| `-s`, `--skills` | 스킬 선탑재 | 반복 업무 자동화 |
| `-c`, `-r` | 세션 복구 | 긴 작업 이어서 진행 |
| `-w`, `--worktree` | 격리 git worktree | 병렬 에이전트 작업 |

## 화면 구성 이해하기

CLI에는 보통 세 가지 영역이 있습니다.

1. 시작 배너: 현재 모델, 작업 디렉터리, 도구, 스킬 요약
2. 대화 스트림: 사용자 입력과 Hermes 응답, 도구 실행 로그
3. 입력 프롬프트: 멀티라인 입력과 자동완성 지원

세션을 재개하면 배너와 입력 프롬프트 사이에 "이전 대화 요약" 패널이 나타납니다. 최근 사용자/어시스턴트 턴, 도구 호출 수만 압축해서 보여 주므로 문맥을 빠르게 되살릴 수 있습니다.

## 상태바 읽는 법

Hermes는 입력창 위에 고정 상태바를 보여 줍니다.

```text
⚕ claude-sonnet-4 │ 12.4K/200K │ [██████░░░░] 6% │ $0.06 │ 15m
```

| 항목 | 의미 |
|------|------|
| 모델명 | 현재 사용 중인 모델 |
| 토큰 사용량 | 사용한 컨텍스트 / 최대 컨텍스트 |
| 컨텍스트 바 | 얼마나 찼는지 시각적으로 표시 |
| 비용 | 세션 추정 비용 |
| 시간 | 세션 경과 시간 |

컨텍스트 사용량 색상은 대략 다음 기준으로 보면 됩니다.

| 색상 | 구간 | 의미 |
|------|------|------|
| 초록 | 50% 미만 | 여유 있음 |
| 노랑 | 50~80% | 슬슬 차기 시작 |
| 주황 | 80~95% | 압축 고려 |
| 빨강 | 95% 이상 | `/compress` 권장 |

:::tip
긴 세션에서 응답이 흔들리기 시작하면 먼저 `/usage`나 `/compress`를 확인하세요. 문제의 상당수는 모델 자체보다 컨텍스트 포화에서 옵니다.
:::

## 주요 키바인딩

| 키 | 동작 |
|----|------|
| `Enter` | 메시지 전송 |
| `Alt+Enter` 또는 `Ctrl+J` | 줄바꿈 |
| `Tab` | 자동완성/고스트 텍스트 수락 |
| `Ctrl+C` | 실행 중인 에이전트 중단 |
| `Ctrl+D` | CLI 종료 |
| `Alt+V` | 클립보드 이미지 붙여넣기 |
| `Ctrl+B` | 음성 모드 녹음 시작/종료 |

멀티라인 입력이 기본적으로 잘 지원되므로, 복잡한 요청은 여러 줄로 구조화해서 주는 편이 좋습니다.

## 슬래시 명령 활용

Hermes CLI는 `/` 입력 시 자동완성 목록을 띄웁니다. 기본 명령과 스킬 명령이 함께 나타납니다.

```text
/help
/model
/tools
/skills browse
/background 이 저장소 테스트를 백그라운드에서 돌려줘
/voice on
/title 인증 리팩터링
```

실전에서 자주 쓰는 조합은 다음과 같습니다.

```bash
# 대화 도중 모델 바꾸기
/model

# 세션 이름 지정
/title 프론트엔드 성능 점검

# 백그라운드 세션 시작
/background pytest와 lint를 돌리고 결과만 요약해줘
```

## 스킬과 워크트리

스킬은 CLI에서 특히 효과가 큽니다. 예를 들어 특정 PR 워크플로나 리뷰 스킬을 선탑재해 대화 시작부터 같은 절차를 반복할 수 있습니다. 또 `--worktree`를 쓰면 현재 저장소와 분리된 Git 작업 디렉터리에서 Hermes를 띄울 수 있어 여러 에이전트를 동시에 돌릴 때 충돌을 줄일 수 있습니다.

| 기능 | 장점 | 주의점 |
|------|------|--------|
| 스킬 선탑재 | 반복 작업 표준화 | 너무 많이 넣으면 초반 문맥이 무거워짐 |
| 워크트리 | 병렬 작업 안전성 | Git 저장소에서만 유효 |
| `--quiet` | 자동화 친화적 출력 | 사람이 보기엔 정보가 적음 |
| `--verbose` | 디버깅에 유리 | 출력이 많아짐 |

## 추천 시작 흐름

처음 CLI를 쓰는 사용자는 아래 순서를 추천합니다.

```bash
hermes
/tools
/model
/title 첫 실습
```

그다음 프로젝트 루트에서 이렇게 물어보면 좋습니다.

```text
이 저장소 구조를 요약하고, 중요한 파일 5개를 추천해줘.
```

CLI는 Hermes의 거의 모든 기능이 만나는 기본 인터페이스입니다. 터미널 사용에 익숙하다면, 이 한 장만 제대로 이해해도 이후의 세션·도구·스킬 문서를 훨씬 쉽게 연결할 수 있습니다.

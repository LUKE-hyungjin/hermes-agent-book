# 34. 번들 스킬 카탈로그

> 공식 문서 기준 Hermes는 설치 시 큰 규모의 번들 스킬 라이브러리를 함께 가져오며, 이 스킬들은 보통 `~/.hermes/skills/`로 복사되어 즉시 사용할 수 있습니다.

## 이 문서를 어떻게 읽어야 하나요?

이 장은 "모든 스킬 본문을 하나하나 번역"하는 페이지가 아니라, **어떤 분야의 스킬이 번들로 들어 있고 언제 꺼내 쓰는가**를 잡아 주는 안내서입니다. 실제 스킬은 각각 독립된 `SKILL.md` 문서이며, 필요할 때만 로드되는 점진적 공개 방식으로 동작합니다.

즉, 스킬 카탈로그는 앱 스토어 목록에 가깝고, 실제 사용법은 개별 스킬 문서에서 확인하는 구조입니다.

## 1. Apple 계열

공식 문서에서 `apple` 카테고리는 macOS 전용 스킬 모음으로 소개됩니다. 공통점은 모두 Apple 생태계 앱과 자동화를 다루며, macOS에서만 로드된다는 점입니다.

| 스킬 | 역할 |
|------|------|
| `apple-notes` | Apple Notes 생성, 검색, 수정 |
| `apple-reminders` | 미리 알림 조회, 추가, 완료 처리 |
| `findmy` | Find My 기반 기기/AirTag 추적 |
| `imessage` | iMessage/SMS 송수신 |

이 묶음은 개발 보조보다 개인 비서형 자동화에 가깝습니다. 맥 사용자라면 "개인 정보 관리자"나 "생활 자동화 비서" 역할을 쉽게 구성할 수 있습니다.

## 2. Autonomous AI Agents

이 카테고리는 스스로 다른 코딩 에이전트를 호출하거나, 여러 에이전트를 병렬로 돌리는 작업에 초점을 둡니다. 말 그대로 "에이전트가 또 다른 에이전트를 다루는" 계층입니다.

| 스킬 | 역할 |
|------|------|
| `claude-code` | Claude Code CLI에 코딩 작업 위임 |
| `codex` | OpenAI Codex CLI에 구현/리뷰 작업 위임 |
| `hermes-agent-spawning` | 별도 Hermes 프로세스를 추가 생성 |
| `opencode` | OpenCode CLI 에이전트에 장기 작업 위임 |

문서 설명을 보면 이들은 단순 호출이 아니라, 기능 구현, 리팩터링, PR 리뷰, 장기 세션 같은 **독립 실행 단위**를 위임하는 데 적합합니다. 다만 전부 외부 CLI 설치나 Git 저장소 전제 같은 조건이 붙을 수 있습니다.

## 3. Data Science

데이터 과학 카테고리는 현재 문서상 `jupyter-live-kernel`이 대표 예시입니다.

| 스킬 | 역할 |
|------|------|
| `jupyter-live-kernel` | 상태를 유지하는 Jupyter 커널 기반 Python 실행 |

이 스킬이 중요한 이유는 Hermes의 기본 셸 실행과 달리, 노트북식 반복 탐색과 중간 결과 보존에 더 잘 맞기 때문입니다. 데이터 정제, 시각화 초안, 실험적 분석처럼 "한 번 실행하고 끝"이 아닌 작업에서 가치가 큽니다.

## 4. Creative

창작 계열 스킬은 결과물이 단순 텍스트를 넘어 시각적 표현으로 나가야 할 때 유용합니다.

| 스킬 | 역할 |
|------|------|
| `ascii-art` | ASCII 아트 생성 |
| `ascii-video` | 비디오/이미지를 ASCII 비주얼로 변환 |
| `excalidraw` | 손그림 스타일 다이어그램 생성 |

특히 `excalidraw`는 아키텍처 다이어그램, 흐름도, 개념도처럼 설명용 산출물이 필요한 상황에서 실용적입니다. 즉, 코딩 에이전트가 코드만 쓰는 것이 아니라 문서와 시각 자료까지 만들어야 할 때 이 카테고리가 살아납니다.

## 5. DevOps와 운영 자동화

문서에 나온 `devops` 대표 스킬은 `webhook-subscriptions`입니다.

| 스킬 | 역할 |
|------|------|
| `webhook-subscriptions` | 외부 이벤트를 받아 Hermes를 자동 활성화 |

이 스킬의 의미는 단순 webhook 등록 도구가 아니라, GitHub·Stripe·CI/CD 같은 외부 시스템 이벤트를 Hermes 작업으로 연결하는 운영 레이어라는 점입니다. 즉, "사용자가 직접 말을 걸어야만 움직이는 에이전트"를 넘어서, 이벤트 기반 자동화로 확장하는 출발점입니다.

## 6. Dogfood와 셀프 운영 보조

문서 말미에는 `dogfood`와 `hermes-agent-setup` 같은 스킬도 보입니다.

| 스킬 | 역할 |
|------|------|
| `dogfood` | 웹 애플리케이션 탐색형 QA와 버그 리포트 작성 |
| `hermes-agent-setup` | Hermes 설치와 설정 절차 보조 |

이 계열은 "Hermes를 더 잘 쓰기 위한 Hermes"에 가깝습니다. 예를 들어 제품 QA를 에이전트에게 맡기거나, 설치 초기에 설정 실수를 줄이는 용도로 활용할 수 있습니다.

## 번들 스킬을 고르는 기준

| 질문 | 적합한 카테고리 |
|------|-----------------|
| 맥 기본 앱과 자동화하고 싶은가? | `apple` |
| 다른 코딩 에이전트에 일을 위임하고 싶은가? | `autonomous-ai-agents` |
| 데이터 분석을 반복적으로 탐색할 것인가? | `data-science` |
| 다이어그램이나 ASCII 결과물이 필요한가? | `creative` |
| 이벤트 기반 운영 자동화를 만들고 싶은가? | `devops` |
| 제품 QA나 설정 보조가 필요한가? | `dogfood` |

:::info
번들 스킬의 가치는 "개수가 많다"가 아니라, 반복되는 절차를 즉시 재사용 가능한 형태로 제공한다는 데 있습니다. 같은 작업을 두 번 이상 설명하게 된다면, 이미 번들 스킬로 해결되는지 먼저 보는 편이 좋습니다.
:::

정리하면 번들 스킬 카탈로그는 Hermes의 숨은 확장 표면입니다. 기본 채팅만 써도 Hermes는 유용하지만, 스킬 카탈로그를 이해하면 Hermes를 단순 응답기가 아니라 운영 절차와 전문 워크플로를 기억하는 시스템으로 바꿀 수 있습니다.

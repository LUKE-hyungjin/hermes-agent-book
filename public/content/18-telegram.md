# 18. Telegram 설정

> Telegram은 Hermes를 가장 빠르게 모바일로 가져오는 채널입니다. BotFather에서 토큰을 만들고, 숫자형 사용자 ID 허용 목록만 맞추면 실전 투입이 가장 쉬운 편에 속합니다.

## Telegram이 잘 맞는 이유

공식 문서는 Telegram 통합이 텍스트, 음성 메모, 이미지, 파일 첨부를 모두 지원한다고 설명합니다. 그래서 개인 비서형 에이전트, 이동 중 점검 봇, 예약 작업 결과 수신 채널에 특히 잘 맞습니다. 메시징 게이트웨이 구조를 그대로 쓰기 때문에, 세션 유지, 메모리, 도구 호출, 음성 응답도 같은 흐름으로 연결됩니다.

## 1단계: BotFather에서 봇 만들기

Telegram 봇에는 `@BotFather`가 발급하는 API 토큰이 필요합니다.

1. Telegram에서 `@BotFather`를 검색합니다.
2. `/newbot`을 보냅니다.
3. 표시 이름을 입력합니다.
4. `bot`으로 끝나는 고유 사용자명을 정합니다.
5. 발급된 토큰을 복사합니다.

토큰은 보통 아래와 비슷한 형태입니다.

```text
123456789:ABCdefGHIjklMNOpqrSTUvwxYZ
```

:::warning
봇 토큰이 유출되면 제3자가 그 봇을 제어할 수 있습니다. 노출이 의심되면 BotFather의 `/revoke`로 즉시 폐기해야 합니다.
:::

## 2단계: 봇 UX 다듬기

BotFather에서 몇 가지 부가 설정을 해 두면 사용자 경험이 좋아집니다.

| 명령 | 역할 |
|------|------|
| `/setdescription` | 시작 전 설명 문구 |
| `/setabouttext` | 프로필 요약 |
| `/setuserpic` | 봇 아바타 |
| `/setcommands` | 채팅 안 명령 메뉴 |
| `/setprivacy` | 그룹 메시지 수신 정책 |

추천 명령 메뉴 예시는 다음 정도면 충분합니다.

```text
help - 도움말 보기
new - 새 대화 시작
sethome - 이 채팅을 홈 채널로 지정
```

## 3단계: Privacy Mode 이해하기

공식 문서는 Telegram 그룹 설정에서 가장 자주 헷갈리는 부분으로 Privacy Mode를 꼽습니다. 기본값이 켜져 있으면 봇은 다음만 볼 수 있습니다.

- `/`로 시작하는 명령
- 봇 메시지에 대한 직접 답장
- 멤버 입장 같은 서비스 메시지
- 봇이 관리자인 채널 메시지

그룹의 일반 대화를 전부 읽게 하려면 Privacy Mode를 끄거나, 해당 그룹에서 봇을 관리자로 승격해야 합니다.

| 선택지 | 장점 | 주의점 |
|------|------|--------|
| Privacy OFF | 그룹 전체 메시지 수신 | 설정 변경 후 봇 재초대 필요 |
| 관리자 승격 | 필요한 그룹만 전체 수신 | 권한 관리 필요 |

:::tip
문서가 제안하듯, 전역 Privacy를 꺼 버리기보다 필요한 그룹에서만 관리자로 올리는 편이 더 보수적이고 운영이 쉽습니다.
:::

## 4단계: 내 사용자 ID 찾기

Hermes는 Telegram 사용자명이 아니라 숫자형 사용자 ID로 접근을 제어합니다. `@userinfobot` 또는 `@get_id_bot`에 메시지를 보내면 바로 확인할 수 있습니다.

```text
123456789
```

이 숫자가 곧 `TELEGRAM_ALLOWED_USERS`에 들어갈 값입니다.

## 5단계: Hermes와 연결하기

가장 쉬운 방법은 설정 마법사입니다.

```bash
hermes gateway setup
```

수동 설정을 원하면 `~/.hermes/.env`에 아래처럼 넣습니다.

```bash
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrSTUvwxYZ
TELEGRAM_ALLOWED_USERS=123456789
```

이후 게이트웨이를 실행합니다.

```bash
hermes gateway
```

## 0.6.0의 Telegram 운영 옵션

0.6.0부터는 Telegram 어댑터를 polling 대신 **webhook mode**로 운영할 수 있습니다. 이 방식은 역프록시 뒤에서 돌릴 때 응답 지연을 줄이고, 프로덕션 배포에 더 잘 맞습니다.

또 그룹에서는 봇 응답 정책을 더 세밀하게 조절할 수 있습니다.

| 그룹 응답 방식 | 의미 |
|------|------|
| 항상 응답 | 그룹 메시지를 더 적극적으로 받음 |
| 멘션될 때만 응답 | 가장 보수적이고 안전한 기본값 |
| regex 트리거 | 특정 패턴이 들어올 때만 반응 |

개인 비서형 봇이라면 멘션 기반이 가장 안전하고, 운영 자동화 봇이라면 webhook mode + 명시적 트리거 패턴 조합이 더 안정적일 수 있습니다.

## 홈 채널 지정

DM이나 그룹 채팅에서 `/sethome`을 실행하면, 해당 채팅이 Cron 결과나 자동화 알림의 기본 수신처가 됩니다. 필요하면 환경 변수로 직접 지정할 수도 있습니다.

```bash
TELEGRAM_HOME_CHANNEL=-1001234567890
TELEGRAM_HOME_CHANNEL_NAME="My Notes"
```

## 운영 팁

| 상황 | 권장 방식 |
|------|-----------|
| 개인 비서 용도 | DM 중심, 허용 사용자 1인 지정 |
| 팀 공유 봇 | 그룹 사용, Privacy 정책 명확화 |
| 예약 작업 결과 수신 | `/sethome` 필수 |
| 음성 메시지 활용 | TTS/STT 설정과 `27-voice` 함께 확인 |

Telegram은 설정 난도가 낮고 모바일 경험이 좋아, 메시징 게이트웨이 입문용으로 가장 추천하기 쉬운 플랫폼입니다.

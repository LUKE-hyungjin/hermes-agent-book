# 01. Hermes Agent란?

> Nous Research가 만든 자율형 AI 에이전트 — 세션을 넘어 기억을 유지하고, 스킬과 컨텍스트를 통해 점점 더 일관되게 일하도록 설계되었습니다.

## 한 줄 정의

**Hermes Agent**는 단순한 챗봇이나 코딩 보조 도구가 아닙니다. 파일을 읽고, 터미널 명령을 실행하고, 웹을 탐색하고, 이메일을 보내는 등 실제 작업을 수행하는 **자율형 AI 에이전트**입니다. 메모리와 스킬, 컨텍스트 파일 덕분에 세션이 바뀌어도 더 일관된 작업 흐름을 이어갈 수 있습니다.

## 일반 챗봇과의 차이

| 특성 | 일반 챗봇 (ChatGPT 등) | Hermes Agent |
|------|----------------------|--------------|
| 파일 접근 | 직접 업로드해야 함 | 필요할 때 파일/디렉터리를 직접 탐색하고 읽기 |
| 작업 범위 | 대화 1회 분량 | 수십~수백 개 파일 동시 처리 |
| 실행 능력 | 텍스트만 출력 | 터미널, 브라우저, API 직접 조작 |
| 기억 | 대화 내에서만 | 세션을 넘어 영구 기억 (Memory) |
| 학습 | 없음 | 경험에서 스킬 자동 생성 (Skills) |
| 메시징 | 웹에서만 | Telegram, Discord, Slack, WhatsApp |

## 핵심 특징

### 1. 자기 개선 루프 (Self-improving Loop)
Hermes는 스킬과 메모리를 통해 유용한 작업 절차를 재사용하고, 필요한 사실을 세션을 넘어 이어 갈 수 있습니다. 다만 모든 작업이 자동으로 스킬이 되는 것으로 이해하기보다, 스킬 시스템을 중심으로 절차 지식을 축적한다고 보는 편이 정확합니다.

### 2. 영구 메모리 (Persistent Memory)
세션이 끝나도 기억이 유지됩니다. 사용자의 선호, 프로젝트 구조, 환경 설정 등을 자동으로 저장하고 다음 세션에서 활용합니다.

### 3. 어디서나 실행 (Deploy Anywhere)
- `$5 VPS`에서도 실행 가능
- GPU 클러스터, 서버리스 인프라 (Daytona, Modal) 지원
- Telegram으로 대화하면서 클라우드 VM에서 작업 수행

### 4. 40+ 내장 도구
파일 읽기/쓰기, 터미널 실행, 웹 탐색, 이메일, 캘린더 등 기본 도구가 내장되어 있고, MCP를 통해 무한 확장 가능합니다.

## 어디서 실행하나요?

| 환경 | 설명 |
|------|------|
| **터미널 (CLI)** | 가장 강력한 기본 환경 |
| **Telegram** | 모바일에서 어디서든 대화 |
| **Discord** | 팀 단위 협업 |
| **Slack / WhatsApp** | 기존 워크플로우에 통합 |
| **API 서버** | 다른 애플리케이션과 연동 |

## 한 줄 요약

> Hermes Agent = "파일·터미널·웹을 직접 다루고, 메모리와 스킬을 통해 세션을 넘어 더 일관되게 일하는 자율형 AI 에이전트"

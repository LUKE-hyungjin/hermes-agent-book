# 25. 비전과 이미지 붙여넣기

> Hermes는 클립보드의 이미지를 CLI에 바로 붙여 넣어 비전 모델에 전달할 수 있습니다. 즉 스크린샷을 저장하고 경로를 넘기는 절차 없이, 터미널 대화 안에서 바로 "이 화면을 분석해줘"가 가능합니다.

## 어떻게 동작하나

공식 문서에 따르면 이미지는 base64 인코딩된 비전 콘텐츠 블록으로 모델에 전달됩니다. 사용 흐름은 단순합니다.

1. 스크린샷이나 브라우저 이미지를 클립보드에 복사
2. `/paste` 또는 단축키로 첨부
3. 질문을 입력하고 Enter
4. 입력창 위에 `[📎 Image #1]` 배지가 표시
5. 전송 시 비전 모델이 이미지를 함께 처리

여러 이미지를 한 번에 붙일 수도 있고, `Ctrl+C`로 첨부 이미지를 모두 비울 수 있습니다. 저장된 이미지는 `~/.hermes/images/` 아래 PNG로 남습니다.

## 붙여넣는 방법

문서는 터미널 환경에 따라 가능한 방법이 다르다고 설명합니다.

### 1. `/paste`

가장 신뢰할 수 있는 방식입니다.

```bash
/paste
```

터미널 키 바인딩에 의존하지 않고 Hermes가 직접 클립보드를 검사하므로, 거의 모든 환경에서 기본 선택지로 권장됩니다.

### 2. `Ctrl+V` 또는 `Cmd+V`

클립보드에 텍스트와 이미지가 함께 들어 있을 때 동작합니다. 다만 이미지 단독 클립보드인 경우 대부분의 터미널은 바이너리 이미지를 직접 붙이지 못하므로 반응하지 않습니다.

### 3. `Alt+V`

많은 터미널에서 Alt 조합은 ESC 시퀀스로 전달되기 때문에 이미지 검사 트리거로 쓸 수 있습니다. 다만 VSCode 통합 터미널은 Alt 조합을 자체 UI에 먼저 잡아먹는 경우가 많아 문서에서도 비권장입니다.

### 4. Linux 전용 `Ctrl+V`

Linux 데스크톱 터미널은 보통 실제 붙여넣기 단축키가 `Ctrl+Shift+V`라서, `Ctrl+V`를 애플리케이션이 직접 받아 클립보드를 검사할 수 있습니다. X11이나 Wayland 접근이 되는 로컬 세션에서만 의미가 있습니다.

## 환경별 호환성 요약

| 환경 | `/paste` | 일반 붙여넣기 | `Alt+V` | 비고 |
|------|----------|---------------|---------|------|
| macOS Terminal / iTerm2 | 좋음 | 좋음 | 좋음 | 기본 경험이 가장 좋음 |
| Linux X11 | 좋음 | 좋음 | 좋음 | `xclip` 필요 |
| Linux Wayland | 좋음 | 좋음 | 좋음 | `wl-clipboard` 필요 |
| WSL2 | 좋음 | 조건부 | 좋음 | `powershell.exe` 사용 |
| VSCode 로컬 터미널 | 좋음 | 조건부 | 낮음 | Alt 조합 제한 |
| SSH/원격 터미널 | 대체로 불가 | 불가 | 불가 | 원격 클립보드 접근 불가 |

문서가 특히 강조하는 점은 SSH나 원격 VSCode 세션에서는 로컬 클립보드에 직접 접근할 수 없다는 것입니다.

## 플랫폼별 준비

### macOS

기본적으로 `osascript`를 사용하므로 별도 준비가 거의 없습니다. 속도를 더 원하면 선택적으로 아래를 설치할 수 있습니다.

```bash
brew install pngpaste
```

### Linux X11

```bash
sudo apt install xclip
```

### Linux Wayland

```bash
sudo apt install wl-clipboard
```

현재 세션이 Wayland인지 확인하는 방법도 문서에 나옵니다.

```bash
echo $XDG_SESSION_TYPE
```

## 언제 유용한가

| 작업 | 효과 |
|------|------|
| UI 스크린샷 설명 | 매우 좋음 |
| 에러 다이얼로그 분석 | 좋음 |
| 디자인 비교 | 좋음 |
| SSH 원격 세션만으로 작업 | 제한 큼 |

:::warning
이미지 전송은 결국 비전 지원 모델이 있어야 제대로 동작합니다. 붙여넣기 자체와 모델의 비전 능력은 별개의 문제입니다.
:::

비전 기능의 포인트는 "이미지 파일 관리"를 줄이고, CLI 대화 안에서 시각 입력을 자연스럽게 섞을 수 있게 만든다는 데 있습니다.

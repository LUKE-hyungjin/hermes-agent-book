# 05. 학습 경로

> Hermes Agent는 기능이 매우 넓기 때문에, "내가 무엇을 하려는지" 기준으로 문서를 읽는 순서를 잡는 것이 가장 빠릅니다.

## 왜 학습 경로가 필요한가요?

Hermes Agent는 단순 CLI 도구가 아닙니다. 터미널 어시스턴트, 메시징 봇, 자동화 엔진, 메모리 시스템, 스킬 플랫폼, 심지어 강화학습 파이프라인까지 아우릅니다. 처음부터 모든 기능을 한꺼번에 익히려 하면 오히려 핵심을 놓치기 쉽습니다. 그래서 이 장에서는 **경험 수준**과 **사용 목적** 두 가지 축으로 읽기 순서를 안내합니다.

## 경험 수준별 추천 순서

아직 설치를 끝내지 않았다면 먼저 `03. 설치 및 설정`을 읽고 실제로 `hermes`가 실행되는지 확인한 뒤 아래 표를 따라가시면 됩니다.

### 처음 10분만 기준으로 보면

초보자는 사실 아래 세 단계만 끝내도 충분합니다.

1. `03. 설치 및 설정`
2. `06. CLI 인터페이스`
3. `07. 설정 관리`

메모리, 메시징, MCP, API 서버는 나중에 필요할 때 붙여도 됩니다.

| 수준 | 목표 | 추천 문서 순서 | 예상 시간 |
|------|------|----------------|-----------|
| 초급 | 설치 후 기본 대화와 설정 감 잡기 | 설치 → CLI → 설정 | 약 1시간 |
| 중급 | 메시징, 메모리, 스킬, 자동화 활용 | 세션 → 메시징 → 도구 → 스킬 → 메모리 → Cron | 2~3시간 |
| 고급 | MCP, 자동화, 확장 구조 이해 | 아키텍처 관점으로 도구 → MCP → Hooks → Delegation → API 서버 | 4시간 이상 |

:::tip
초반에는 "모든 기능 이해"보다 "하나의 실제 흐름 완성"이 중요합니다. 예를 들어 CLI 사용자라면 `06. CLI 인터페이스`와 `13. 컨텍스트 파일`까지만 먼저 익혀도 바로 생산성을 낼 수 있습니다.
:::

## 사용 목적별 읽기 지도

문서를 순서대로 읽는 것보다, 자신이 원하는 결과에서 역으로 들어오는 편이 더 빠른 경우가 많습니다.

### 1. CLI 코딩 어시스턴트를 원할 때

터미널 안에서 코드 읽기, 수정, 명령 실행까지 맡기고 싶다면 다음 순서를 추천합니다.

```text
03 설치 → 06 CLI → 07 설정 → 13 컨텍스트 파일 → 25 코드 실행 → 32 CLI 레퍼런스
```

핵심은 프로젝트 루트에서 Hermes를 실행하고, `AGENTS.md`나 `.hermes.md` 같은 컨텍스트 파일로 저장소 규칙을 알려주는 것입니다.

### 2. Telegram/Discord 봇을 만들고 싶을 때

```text
07 설정 → 17 메시징 개요 → 18 Telegram 또는 19 Discord → 10 보안 → 27 음성
```

이 경로는 게이트웨이 구조와 접근 제어를 먼저 이해해야 실수가 적습니다. 특히 공개 서버에서 쓸 계획이라면 `10-security`를 건너뛰면 안 됩니다.

### 3. 자동화와 스케줄링이 목적일 때

```text
21 Cron → 22 Hooks → 23 Delegation → 11 메모리
```

예약 작업만 필요한지, 이벤트 기반 후크가 필요한지, 아니면 서브에이전트 병렬화가 필요한지에 따라 설계가 갈립니다.

### 4. 확장 기능을 만들고 싶을 때

| 목표 | 먼저 읽을 문서 | 이유 |
|------|----------------|------|
| 내장 도구 활용 | `15-tools` | 어떤 도구셋이 이미 있는지 파악 |
| 외부 서버 연결 | `16-mcp` | MCP 기반 확장 구조 이해 |
| 재사용 워크플로 구축 | `12-skills` | SKILL.md 형식과 로딩 방식 학습 |
| 조직용 연결 | `31-api-server` | 외부 UI나 앱과 연결 |

## 추천 학습 시나리오

아래는 한국 사용자 기준으로 가장 많이 쓰이는 세 가지 시나리오입니다.

### 개인 개발자

- `06-cli`로 기본 대화
- `08-sessions`로 이어서 작업
- `13-context-files`로 프로젝트 규칙 주입
- `25-code-execution`으로 반복 작업 자동화

### 팀 봇 운영자

- `17-messaging-overview`
- `19-discord` 또는 `18-telegram`
- `10-security`
- `21-cron`
- `22-hooks`

### 파워유저 / 플랫폼 빌더

- `07-configuration`
- `09-providers`
- `15-tools`
- `16-mcp`
- `23-delegation`
- `31-api-server`

## 빠르게 감 잡는 실습 루트

아래 순서대로 실제로 따라 하면 대부분의 핵심 기능이 연결됩니다.

```bash
hermes
hermes model
hermes chat -q "내 작업 디렉터리를 요약해줘"
```

이후 프로젝트 루트에 `AGENTS.md`를 두고 다시 실행해 보면, Hermes가 같은 저장소에서도 훨씬 더 일관된 답변을 내는 것을 확인할 수 있습니다.

## 무엇을 나중으로 미뤄도 되나요?

| 지금 안 읽어도 되는 문서 | 이유 |
|-------------------------|------|
| `16-mcp` | 외부 툴 서버를 붙일 때만 필요 |
| `27-voice` | 음성 입출력이 필요할 때만 |
| `30-docker` | 격리 실행이 필요할 때 |
| `35-skills-catalog` | 내장 스킬을 적극적으로 활용할 때 |

:::info
문서를 모두 읽고 시작하는 방식보다, 현재 문제를 해결하는 데 필요한 4~5장만 먼저 읽고 실제로 써본 뒤 나머지를 확장하는 방식이 훨씬 효율적입니다.
:::

학습 경로의 핵심은 완독이 아니라 **적절한 순서로 실전 기능을 연결하는 것**입니다. Hermes는 기능이 많지만, 처음부터 필요한 축만 잡으면 진입 장벽이 생각보다 낮습니다.

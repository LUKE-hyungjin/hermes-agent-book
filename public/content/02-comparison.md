# 02. 다른 에이전트와의 비교

Hermes Agent, OpenClaw, Claude Code는 모두 "에이전트"로 불리지만 실제 초점은 조금 다릅니다. 이 장은 **Hermes 공식 문서 요약이라기보다 공개 제품 설명과 현재 생태계 흐름을 바탕으로 한 비교 메모**로 읽는 편이 맞습니다.

## 한눈에 보는 비교

| 항목 | Hermes Agent | OpenClaw | Claude Code |
| --- | --- | --- | --- |
| 기본 성격 | 자기호스팅형 범용 에이전트 | 멀티채널 개인 AI 어시스턴트 플랫폼 | 터미널 중심 코딩 에이전트 |
| 공개 문서에서 특히 강조하는 점 | 지속 메모리, 스킬, 장기 세션 운용 | 게이트웨이, 메시징 채널 통합, 로컬 우선 운영 | 코드 작성·수정·디버깅·자동화 |
| 주 사용 환경 | 서버, VPS, 로컬 머신 | 게이트웨이가 돌아가는 로컬/서버 환경 | 개발자의 터미널, IDE, CI |
| 접근 방식 | "계속 살아 있는" 개인 에이전트에 가깝습니다 | 여러 앱에서 불러 쓰는 항상 켜진 어시스턴트에 가깝습니다 | 현재 코드베이스를 대상으로 바로 작업하는 도구에 가깝습니다 |
| 코딩 작업 적합성 | 가능하지만 코딩만을 위한 도구로 보이진 않습니다 | 가능하지만 본질은 채널/세션 운영 쪽이 더 강합니다 | 가장 직접적인 선택지입니다 |
| 멀티플랫폼 메시징 | 강하게 지원합니다 | 매우 강하게 지원합니다 | 주 초점은 아닙니다 |
| 개방성 | 오픈소스, 자기호스팅 중심 | 오픈소스, 자기호스팅 중심 | Anthropic 제공 도구 중심 |
| 선택이 잘 맞는 경우 | 장기 기억과 개인화가 중요한 경우 | 여러 채널에서 하나의 어시스턴트를 쓰고 싶은 경우 | 로컬 코드 작업 생산성이 가장 중요한 경우 |

## 각 도구의 특징

### Hermes Agent

- 공개 소개에서는 "시간이 갈수록 더 잘 아는 에이전트"라는 점을 전면에 둡니다.
- 지속 메모리, 자동 스킬 생성, 스케줄링, 멀티플랫폼 접속 같은 기능이 함께 묶여 있습니다.
- 단순한 코딩 보조기라기보다, 오래 실행되며 사용자의 프로젝트와 습관을 축적하는 개인 에이전트에 더 가깝습니다.
- 따라서 "한 번의 코드 수정"보다 "계속 관계를 유지하는 에이전트"가 필요할 때 더 어울립니다.

### OpenClaw

- OpenClaw 문서는 이를 자기 장비에서 운영하는 개인 AI 어시스턴트이자 게이트웨이로 설명합니다.
- 핵심은 WhatsApp, Telegram, Slack, Discord 같은 채널을 하나의 게이트웨이로 묶는 구조입니다.
- 코딩 에이전트와 연결하는 기능도 있지만, 제품의 중심은 코드 편집 자체보다 세션 라우팅, 채널 연결, 장기 운영에 더 가까워 보입니다.
- 그래서 "어디서나 메시지로 접근 가능한 AI"가 필요할 때 특히 강점이 드러납니다.

### Claude Code

- Anthropic은 Claude Code를 터미널에 사는 코딩 도구로 설명합니다.
- 자연어로 기능 구현, 버그 수정, 코드 탐색, git 작업 자동화를 수행하는 흐름이 명확합니다.
- 파일 편집, 명령 실행, 외부 도구 연결까지 코딩 워크플로 안에서 바로 이어지는 점이 장점입니다.
- 반대로, 개인 메시징 허브나 항상 켜진 개인 에이전트라는 관점은 상대적으로 덜 강조됩니다.

## 어떤 도구를 선택할까요?

코드 작업이 중심이라면 Claude Code가 가장 단순한 출발점으로 느껴질 수 있습니다. 다만 이는 절대적인 우열이라기보다 제품 초점의 차이로 이해하는 편이 맞습니다.

항상 켜져 있고, 기억을 쌓고, 여러 채널에서 계속 이어지는 개인 에이전트가 필요하다면 Hermes Agent 쪽이 더 자연스럽습니다. 반대로 여러 메신저와 채널을 한데 묶는 운영 허브가 우선이라면 OpenClaw가 더 잘 맞을 수 있습니다.

즉, 세 도구를 단순히 "누가 더 좋다"로 비교하기보다는 다음처럼 보는 편이 정확합니다. 장기 기억과 자기개선이 중요하면 Hermes Agent, 채널 통합과 게이트웨이 운영이 중요하면 OpenClaw, 개발 생산성이 핵심이면 Claude Code를 먼저 검토하는 방식입니다.

# 02. 다른 에이전트와의 비교

> "최고의 도구는 하나가 아니다. 상황에 맞는 도구를 고르고, 필요하면 함께 쓰면 된다."

AI 에이전트 시장이 빠르게 성장하면서, 각 도구마다 명확한 철학과 강점이 있습니다. 이 문서에서는 **Hermes Agent**, **OpenClaw**, **Claude Code** 세 가지를 비교하고, **acpx**를 통해 이들을 연결하는 방법까지 살펴봅니다.

## 한눈에 보는 비교

| 비교 항목 | Hermes Agent | OpenClaw | Claude Code |
|-----------|-------------|----------|-------------|
| **개발사** | Nous Research | 오픈소스 커뮤니티 | Anthropic |
| **핵심 목적** | 자율형 범용 에이전트 | 코드 생성/분석 CLI | 코드 이해/생성 에이전트 |
| **자율성 수준** | 높음 — 스스로 판단하고 실행 | 중간 — 프롬프트 기반 실행 | 중간 — 프롬프트 기반 실행 |
| **파일/터미널 조작** | 기본 내장 (40+ 도구) | 지원 | 지원 |

| **MCP 지원** | 네이티브 지원 (내장 클라이언트) | 네이티브 지원 | 네이티브 지원 |
| **에이전트 간 통신** | acpx로 Codex/Claude/OpenClaw 호출 | 없음 | 없음 |

| **확장성** | 스킬 + MCP + acpx | MCP 플러그인 | MCP 서버 |
| **시각/음성** | 이미지 분석, TTS/STT 내장 | 제한적 | 이미지 분석 지원 |
| **호출 위임** | 서브에이전트, delegate_task | 없음 | 없음 |

## 각 도구의 특징

### Hermes Agent

스스로 학습하고 발전하는 자율형 AI 에이전트입니다. 작업을 완료하면 그 과정을 스킬로 자동 저장하고, 세션이 바뀌어도 사용자를 기억합니다. 40개 이상의 내장 도구로 파일 시스템, 터미널, 브라우저, 이메일, 캘린더 등을 직접 조작하며, Telegram과 Discord 등 메시징 플랫폼에서도 실행할 수 있습니다. 단순한 코딩 도구가 아니라 **일상과 개발 전반을 아우르는 비서 역할**을 합니다.

### OpenClaw

오픈소스 기반의 AI CLI로, 코드 생성과 분석에 특화되어 있습니다. MCP(Model Context Protocol)를 네이티브로 지원하여 다양한 도구와 서비스를 연결할 수 있으며, 커뮤니티 주도로 빠르게 발전하고 있습니다. 가볍고 유연한 구조가 장점이며, 개발자가 자신의 워크플로우에 맞게 커스터마이징하기 좋습니다.

### Claude Code

Anthropic에서 공식 개발한 CLI 에이전트로, Claude 모델의 강력한 코드 이해력을 바탕으로 동작합니다. 코드베이스 분석, 리팩토링, PR 리뷰 등에서 뛰어난 성능을 보이며, Anthropic 생태계와 깊게 통합되어 있습니다. CLAUDE.md 파일을 통해 프로젝트 컨텍스트를 관리할 수 있습니다.

## 고급 기능: acpx로 에이전트 간 통신

### acpx란?

**acpx** (Agent Communication Protocol eXchange)는 Hermes Agent가 다른 AI 에이전트와 통신할 수 있게 해주는 브릿지 프로토콜입니다. ACP(Agent Communication Protocol)를 기반으로 하며, 하나의 에이전트 안에서 여러 AI의 강점을 조합할 수 있습니다.

### 작동 방식

```
사용자 → Hermes Agent → acpx → 대상 에이전트 (Codex / Claude Code / OpenClaw)
                                     ↓
                               작업 수행
                                     ↓
Hermes Agent ← acpx ← 결과 수신 ← 대상 에이전트
```

Hermes가 작업을 받으면, 필요에 따라 가장 적합한 에이전트에게 작업을 위임합니다. 결과는 다시 Hermes로 돌아와 사용자에게 전달됩니다.

### 왜 중요한가?

각 에이전트마다 강점이 다릅니다:

- **Codex** — 대량 코드 생성, 배치 작업, 병렬 이슈 수정에 강함
- **Claude Code** — 심층 코드 리뷰, 리팩토링, 복잡한 버그 분석에 강함
- **OpenClaw** — 경량 분석, MCP 기반 워크플로우에 강함

예를 들어, "이 프로젝트를 리팩토링해줘"라는 요청이 들어오면:

1. Hermes가 작업을 분석
2. 아키텍처 설계는 직접 수행
3. 세부 구현은 Codex에게 위임
4. 코드 리뷰는 Claude Code에게 요청
5. 모든 결과를 통합하여 사용자에게 전달

**한 명의 비서가 여러 전문가를 조율하는 것과 같습니다.**

### 현재 지원 대상

| 대상 에이전트 | 주요 용도 |
|-------------|----------|
| OpenAI Codex | 대량 코드 생성, 배치 작업, 병렬 구현 |
| Claude Code | 심층 코드 리뷰, 리팩토링, 분석 |
| OpenClaw | 경량 분석, MCP 워크플로우 |

## 어떤 도구를 선택할까요?

### Hermes Agent가 적합한 경우

- 세션을 넘어 기억하고 학습하는 에이전트가 필요할 때
- Telegram이나 Discord에서 모바일로 작업하고 싶을 때
- 이메일, 캘린더, 파일 등 일상 업무까지 자동화하고 싶을 때
- 정기적인 예약 작업(크론)이 필요할 때
- 하나의 에이전트가 모든 걸 조율하게 하고 싶을 때

### Claude Code가 적합한 경우

- 코드베이스를 깊이 이해하고 분석해야 할 때
- Anthropic 생태계(Claude 모델)에 이미 익숙할 때
- 리팩토링과 코드 품질 개선에 집중할 때

### OpenClaw가 적합한 경우

- 가볍고 유연한 CLI 도구가 필요할 때
- MCP 네이티브 워크플로우를 구축할 때
- 오픈소스 커뮤니티 기반 도구를 선호할 때

### 가장 좋은 선택: 함께 쓰기

Hermes Agent의 acpx를 통해 세 도구를 모두 연결하면, **각자의 강점만 가져오는 최적의 워크플로우**를 만들 수 있습니다. Hermes가 중앙 허브 역할을 하고, 작업의 성격에 따라 최적의 에이전트에게 위임합니다.

> "도구는 단일점이 아니라 생태계다. 연결된 도구가 가장 강하다."

# 25. 코드 실행

> `execute_code`는 에이전트가 Python 스크립트를 잠깐 작성해 Hermes 도구를 프로그래밍적으로 호출하게 하는 기능입니다. 여러 도구 호출과 중간 가공을 한 번의 LLM 턴으로 압축할 때 진가가 납니다.

## 왜 필요한가

일반적인 도구 호출은 "한 번 호출하고 결과를 읽고, 다시 호출"하는 흐름이라 반복 처리나 조건 분기가 많아지면 컨텍스트를 빠르게 잡아먹습니다. 문서는 `execute_code`가 이 문제를 줄이기 위해 설계됐다고 설명합니다.

스크립트는 에이전트 호스트에서 샌드박스된 자식 프로세스로 실행되고, Hermes와는 Unix domain socket RPC로 통신합니다.

## 동작 방식

흐름은 다음과 같습니다.

1. 에이전트가 `from hermes_tools import ...` 형식의 Python 스크립트를 씁니다.
2. Hermes가 RPC 함수가 들어 있는 `hermes_tools.py` 스텁을 생성합니다.
3. Hermes가 Unix domain socket과 RPC 리스너 스레드를 엽니다.
4. 스크립트가 자식 프로세스에서 실행되고, 도구 호출은 소켓을 통해 Hermes로 돌아옵니다.
5. 최종적으로는 스크립트의 `print()` 출력만 LLM에 반환됩니다.

즉 중간 검색 결과나 파일 읽기 결과는 모델 문맥창에 직접 쌓이지 않습니다.

## 샌드박스에서 쓸 수 있는 도구

문서에 명시된 도구는 다음과 같습니다.

- `web_search`
- `web_extract`
- `read_file`
- `write_file`
- `search_files`
- `patch`
- `terminal` (포그라운드 전용)

예시는 아래처럼 단순합니다.

```python
from hermes_tools import web_search, web_extract

results = web_search("Python 3.13 features", limit=5)
for r in results["data"]["web"]:
    content = web_extract([r["url"]])
print("summary")
```

## 언제 자동으로 쓰게 되나

문서는 다음 조건에서 에이전트가 `execute_code`를 선택한다고 정리합니다.

- 도구 호출이 3회 이상 필요할 때
- 중간 가공 로직이나 조건 분기가 있을 때
- 결과 집합을 반복문으로 순회해야 할 때

핵심 이점은 토큰 절약입니다. 중간 결과는 문맥에 안 들어오고 최종 `print()`만 돌아오므로, 장문의 검색 결과를 직접 대화에 계속 쌓는 것보다 훨씬 효율적입니다.

## 대표 사례

| 사례 | 설명 |
|------|------|
| 데이터 처리 파이프라인 | 여러 설정 파일을 찾아 일부만 발췌 |
| 다단계 웹 조사 | 검색 후 본문 추출, 발췌, 요약 |
| 대량 파일 수정 | 패턴 검색 후 일괄 패치 |
| 빌드·테스트 파싱 | 테스트 실행 후 통계 추출 |

예를 들어 문서에는 `database`가 들어간 YAML 파일을 찾고 일부만 JSON으로 출력하거나, `old_api_call(`을 `new_api_call(`로 일괄 교체하는 예시가 나옵니다.

## 자원 제한

문서에 적힌 기본 제한은 다음과 같습니다.

| 자원 | 제한 | 비고 |
|------|------|------|
| 실행 시간 | 5분 | 이후 SIGTERM, 5초 뒤 SIGKILL |
| stdout | 50KB | 초과 시 잘림 표시 |
| stderr | 10KB | 실패 시 디버깅용 포함 |
| 도구 호출 수 | 50회 | 초과 시 오류 |

즉 이 기능은 "작은 자동화 스크립트"에는 적합하지만, 장시간 실행하는 배치 작업이나 대규모 파이프라인을 대신하는 용도는 아닙니다.

:::tip
`execute_code`는 파이썬을 잘 쓰기 위한 기능이 아니라, LLM의 컨텍스트를 덜 쓰면서 복잡한 도구 흐름을 묶기 위한 기능입니다.
:::

## 언제 쓰지 말아야 하나

| 상황 | 적합성 |
|------|--------|
| 단일 도구 호출 1~2회 | 낮음 |
| 중간 결과를 사람이 직접 검토해야 함 | 낮음 |
| 반복·분기·집계가 많은 작업 | 높음 |

이 기능을 이해할 때 가장 중요한 포인트는 "스크립트 실행"보다 "중간 상태를 문맥 바깥으로 밀어낸다"는 설계 목적입니다.
